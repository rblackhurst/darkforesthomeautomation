STEP 01

Hardware Preparation

Inspect the NUC before doing anything else. A damaged unit found after hours of configuration work costs you the whole evening.

  1. 1
    Unbox the Intel N100 NUC. Inspect for physical damage — corners, ports, bottom panel.
  2. 2
    Open the bottom panel. Confirm RAM is seated — 8GB SO-DIMM. Confirm primary SSD installed — 256GB M.2.
  3. 3
    Check whether the second M.2 or SATA slot is available and empty. Document YES or NO on the ownership sheet and install worksheet.
  4. 4
    Insert the 32GB mini USB drive into a rear USB port. This is the permanent local backup drive — rear port keeps it out of the way and less likely to be bumped. Do not use a front port.
  5. 5
    Seat the Sonoff Zigbee Dongle Plus in a USB port. Front port is fine for the dongle.
  6. 6
    Enter BIOS — press F2 at the splash screen on most Intel NUCs. Update firmware if a newer version is available. Set boot order so USB is first. Disable Secure Boot. Save and exit.
⚠
Secure Boot will block HAOS from booting. This is the most common reason for a failed first boot — always confirm it is off before moving on.

STEP 05

Backup Configuration

Configure backups before installing any add-ons. If anything goes wrong during setup you want a clean restore point.

Settings › System › Storage
  1. 1
    Confirm the 32GB mini USB drive is visible under Storage. If it does not appear, unplug and reseat it.
  2. 2
    Go to Settings › System › Backups. Set backup location to the USB drive. Set schedule to weekly.
  3. 3
    Install the backup retention automation. Go to Settings › Automations › Create Automation, switch to Edit in YAML, and paste:
Backup Retention Automation — YAML 
alias: Backup Retention - Keep 16 Most Recent
description: Runs after each automated backup, removes backups beyond 16 copies
trigger:
  - platform: event
    event_type: backup_successful
action:
  - service: backup.delete
    data:
      keep_last: 16
mode: single
ℹ
A full HAOS backup for a standard residential install runs ~200–500MB. Sixteen backups uses at most 8GB on a 32GB drive — no storage concern.
  1. 4
    Take the first manual backup now. Navigate to Settings › System › Backups › Create Backup. Select Full backup. Wait for it to complete. Confirm it appears in the backup list and is written to the USB drive.

Capture the Backup Encryption Key

HAOS encrypts all backups. The encryption key is required to restore from a backup — without it, encrypted backups are unrecoverable. Capture and store it now.

  1. 5
    Navigate to Settings › System › Backups › Manage Encryption Key. Copy the full key.
Backup Enc. Key ⇄
Syncs to Client Record automatically. Also needs to go on the USB drive and ownership sheet — see steps below.
  1. 6
    Save the key to a plain text file named backup_key.txt and copy it to the 32GB mini USB drive in the rear port of the NUC. This file travels with the hardware.
  2. 7
    Record the key in the DFHA customer portal record for this client. It will be included in the ownership sheet at the end of the install — the Client Record (⬡ button) captures it automatically once you fill in the field above.
⚠
Without this key, encrypted backups cannot be restored. If the NUC dies and you do not have the key, the client's configuration is unrecoverable. Treat it like a password.
ℹ
Manual backup shortcut for later steps: Settings › System › Backups › Create Backup → Full backup. After each major section, label the backup with a short descriptor (e.g. post-mosquitto, post-adguard).

STEP 07

Mosquitto MQTT Broker

Mosquitto is the MQTT broker — the message bus that Zigbee2MQTT and other services use to communicate with HAOS. It must be running and connected before Zigbee2MQTT is configured. The mqtt_admin account was created in Step 4. All MQTT configuration is done in this step.

Install and Start Mosquitto

Settings › Apps › + Install app › Search "Mosquitto broker"
  1. 1
    Install the Mosquitto broker add-on. Do not start it yet.
  2. 2
    Go to the Configuration tab. The default configuration is shown below — do not paste anything into the YAML editor. The logins section in the config tab is for legacy local-only accounts. Because you created mqtt_admin as a standard HAOS user in Step 4, Mosquitto will authenticate it automatically. Leave the configuration at its defaults:
Mosquitto Configuration — leave at defaults 
logins: []
customize:
  active: false
  folder: mosquitto
certfile: fullchain.pem
keyfile: privkey.pem
require_certificate: false
ℹ
Mosquitto uses HAOS user accounts for authentication automatically. The mqtt_admin account you created in Step 4 is available to Mosquitto without any additional configuration in the add-on.
  1. 3
    Start the add-on. Check the Log tab — confirm it shows started with no errors. Set Start on boot and Watchdog both ON.

Connect the MQTT Integration

Settings › Devices & Services › Add Integration › MQTT
  1. 4
    After starting Mosquitto, HAOS will show it as a discovered integration at the top of Devices & Services. Click Configure on the Mosquitto broker tile — or manually add the MQTT integration and fill in:
MQTT Integration — Broker Settings 
Broker:    core-mosquitto
Port:      1883
Username:  mqtt_admin
Password:  MQTT[Last4]    (e.g. MQTTSmit — set in Step 4)
ℹ
core-mosquitto is the internal HAOS hostname for the Mosquitto add-on. Always use this instead of an IP address — it resolves correctly regardless of network changes.
  1. 5
    Confirm the MQTT integration shows as connected in Devices & Services.
  2. 6
    The MQTT integration Configure dialog will show __**password_not_changed**__ in the password field. This is normal — it is the supervisor's internal service token, not a broken credential. Do not try to change it. Z2M authenticates separately using the mqtt_admin account in the Mosquitto logins, which is correct.
  3. 7
    Take a manual backup. Label it post-mosquitto.

ℹ
